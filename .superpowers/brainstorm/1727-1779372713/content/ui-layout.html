A — Structured Chat + Tables

Startup panel, spinner while MCP tools run, Spectre tables for position results, color-coded roles (You/Assistant). Highest visual impact. Best for demo and blog screenshots.

Pros

  • Visually impressive
  • Tables great for position data
  • Clear tool call status

Cons

  • More code changes
  • Table format depends on response shape

B — Rule-Separated Panels

Horizontal rules between turns, colored speaker labels, spinner for tool calls, response text in a left-bordered panel. Clean and readable without heavy chrome.

Pros

  • Clean, minimal
  • Easy to implement
  • Scales well to long responses

Cons

  • Less visual impact
  • No structured tables

C — Full Panel Borders

Each message (user and assistant) wrapped in a Spectre Panel with a colored border and role label. Chat-app style. Startup in a header panel. Tool calls shown inline.

Pros

  • Chat-app feel
  • Clear visual separation per turn

Cons

  • Verbose for long sessions
  • More rendering overhead
